# 内容同步矩阵 SOP (掘金 / 知乎 / 微信公众号)

> 建立于：PET-112 (2026-05-25)
> 维护者：gemini 1号
> 目标：打通国内主流长文内容平台，实现博客内容二次分发，引流至官网。

---

## 一、 平台定位与分发策略

| 平台 | 定位 | 内容偏好 | 转化目标 |
| :--- | :--- | :--- | :--- |
| **掘金 (Juejin)** | 开发者社区 | 技术实践、源码分析、工具分享 | 个人技术品牌、官网流量 |
| **知乎 (Zhihu)** | 综合问答与专栏 | 深度见解、行业分析、方法论 | SEO 权重、专业信任度 |
| **微信公众号** | 订阅制私域 | Newsletter 摘要、深度长文 | 用户留存、邮件列表订阅 |

- [ ] 文章已在 [peterclaw.com](https://peterclaw.com) 发布。

## 三、 各平台分发步骤

### 3.1 掘金 (Juejin)

掘金分发已升级至 **v2.0** 独立 SOP，详见 [`docs/juejin-distribution-sop-v2.md`](./juejin-distribution-sop-v2.md)。

**关键摘要**：
- **双版本策略**：博客「完整版 + 系列上下文」 vs 掘金「实操精简版」（三段式结构）。
- **首发权**：博客必须先发布并被 Google 索引，掘金版方可上线。
- **重复率控制**：掘金改写稿与博客原文文本重复率 < 40%。
- **发布检查**：图片上传掘金图床、代码块测试高亮、站外链接 ≤ 3 个且置文末。
- **互动维护**：发布后 24h 内集中回复技术问题，记录高频问题反哺选题。

### 3.2 知乎 (Zhihu)
1. **格式：** 支持 Markdown 导入或富文本。
2. **操作：** 使用知乎自带的“导入 Markdown”工具，或直接粘贴并校对格式。
3. **脚注：** 在末尾添加：`> 首发于 [peterclaw.com](https://peterclaw.com)`。
4. **发布：** 发布为“文章”，并关联至个人专栏。

### 3.3 微信公众号
1. **格式：** 仅支持富文本。
2. **工具：** 使用 [Md2All](https://md.pynote.net/) 或 [Markdown Nice](https://mdnice.com/) 将 Markdown 转换为公众号样式。
3. **操作：** 复制渲染后的富文本粘贴至公众号后台。
4. **链接：** 底部“阅读原文”链接务必指向官网文章 URL。
5. **互动：** 在文末引导读者前往官网订阅 Newsletter。

## 四、 阻塞处理协议

若在执行过程中遇到以下情况，请立即停止并按协议处理：
- **需要人工授权：** 如扫码登录、短信验证码、实名认证、人脸识别等。
- **账号未建立：** 需要人类负责人注册账号。
- **权限受限：** 如无法添加外链或发布文章。

**处理流程：**
1. 将当前 issue 状态改为 `blocked`。
2. 创建子 issue，标题为 `【待你行动】账号授权：[平台名称]`。
3. 分配给 Workspace Owner。
4. 在主 issue 评论说明阻塞原因。
