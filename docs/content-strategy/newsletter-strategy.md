## 1. 定位与价值主张

**PeterClaw Newsletter** 是 PeterClaw 网站的邮件订阅频道，面向关注 AI 工具与独立开发者实践的读者。

- **定位**：AI 工具周报 + 独立开发者洞察
- **口号**："每周一次，看见 AI 如何真正改变开发工作流"
- **差异化**：
  - 不追热点，只选经过真实项目验证的工具与观点
  - 内容与博客/评测形成呼应，引流回站内深度阅读
  - 公开记录 Newsletter 的迭代过程（内容即产品）

---

## 2. 发送节奏

| 项目 | 设定 |
|------|------|
| **频率** | 每两周一期（Bi-weekly） |
| **发送日** | 隔周五上午 10:00 CST |
| **首期** | 2026-05-30（Vol.1） |
| **工具** | [Buttondown](https://buttondown.com) |
| **订阅入口** | 网站首页 `/zh/` 与 `/en/` 的 EmailSignup 组件 |

### 为什么是两周一期？

1. **内容密度**：网站保持每周 2-3 篇更新，两期之间可积累足够的高质量素材。
2. **读者耐受**：中文开发者订阅箱普遍拥挤，双周报比周刊更易被完整阅读。
3. **运营可持续**：单期筹备时间约 2-3 小时，不会挤占核心内容生产。

## 3. 内容结构（每期固定模板）

```
┌─────────────────────────────────────────┐
│  PeterClaw Newsletter · Vol.X           │
│  发刊词 / 主编寄语（1-2 句）              │
├─────────────────────────────────────────┤
│  🎯 本期亮点（2-3 条）                    │
│     → 一句话摘要 + 站内外链接              │
├─────────────────────────────────────────┤
│  🛠 精选 AI 工具动态（3-5 条带简评）       │
│     → 工具名 + 更新/发布 + 一句评价        │
├─────────────────────────────────────────┤
│  📖 本周 peterclaw.com 新内容导读         │
│     → 最新 2-3 篇文章的导语 + 直达链接     │
├─────────────────────────────────────────┤
│  🔭 下期预告                              │
│     → 1 个即将发布的选题或调研方向         │
├─────────────────────────────────────────┤
│  页脚：退订 / 反馈 / 社交媒体             │
└─────────────────────────────────────────┘
```

### 内容 sourcing 渠道

- **站内新内容**：博客更新（AI 日记、GitHub 深潜、工具评测）
- **工具动态**：Cursor、Windsurf、Claude Code、Bun、n8n 等已覆盖工具的 release note
- **独立开发者洞察**：AI 小队公开协作日志中的经验与教训
- **社区信号**：即刻、X（Twitter）、GitHub Trending 中的高信噪比讨论

## 4. 视觉与格式规范

### HTML 邮件要求

- **移动端优先**：最大宽度 600px，字体不小于 14px
- **暗色兼容**：Buttondown 默认支持暗色模式，避免使用纯黑背景
- **品牌色**：主色 `#6366f1`（Indigo-500），用于标题与链接
- **CTA 按钮**：圆角 8px，主色填充，白色文字
- **图片**：尽量使用文字替代，必要时以 `max-width: 100%` 约束

### 多语言策略

首期先以 **中文** 为主（订阅入口目前主要面向中文读者），Vol.3 后视订阅者构成评估是否增加英文版。

## 5. 数据指标与成功标准

| 指标 | 首期目标 | 3 期后目标 | 6 期后目标 |
|------|---------|-----------|-----------|
| 订阅人数 | 20 | 100 | 300 |
| 打开率 | ≥ 35% | ≥ 40% | ≥ 45% |
| 点击率 | ≥ 8% | ≥ 10% | ≥ 12% |
| 退订率 | ≤ 5% | ≤ 3% | ≤ 2% |

### 首期核心目标

> **验证订阅者响应率** —— 不追求完美，只验证"有人愿意每周花 3 分钟读我们的内容"。

---

## 6. 运营工作流（SOP）

### 每期生产流程

```
T-7 日  ━━  内容收集：从博客更新、工具 release、社区讨论中筛选素材
T-5 日  ━━  选题会：确定本期亮点与工具动态清单（AI 小队内部评审）
T-3 日  ━━  初稿撰写：按模板输出 Newsletter Markdown + HTML
T-2 日  ━━  内部审校：检查链接、排版、移动端预览
T-1 日  ━━  导入 Buttondown：粘贴 HTML，设置发送时间，预览测试
T-0 日  ━━  定时发送 + 截图存档 + 更新本文档的"已发送列表"
```

### 存档规范

- 每期 HTML 源文件保存于 `docs/content-strategy/newsletter-vol-XX.html`
- 发送后截图保存于 `docs/content-strategy/assets/newsletter-vol-XX-preview.png`
- 发送数据（打开率、点击率）记录在本文档的"已发送列表"中

## 8. 迭代计划

- **Vol.1-2**：固定模板跑通，收集读者反馈
- **Vol.3-4**：根据数据调整内容比例（如增加/减少工具动态篇幅）
- **Vol.5+**：引入读者问答或投票环节，提升互动率
