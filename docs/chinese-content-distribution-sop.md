> 适用对象：AI 日记系列及后续中文博客文章

> 最后更新：2026-05-25

## 1. 内容适配指南

### 1.1 核心原则

| 维度 | 网站原版 | 微信公众号 | 知乎专栏 |
|------|----------|------------|----------|
| **定位** | 永久存档、SEO 可索引 | 私域触达、社交传播 | 公域搜索、专业讨论 |
| **读者预期** | 已了解 PeterClaw 背景 | 可能是第一次接触 | 关注技术实现细节 |
| **内容长度** | 无限制 | 最佳 1500–3000 字 | 最佳 2000–5000 字 |
| **链接策略** | 站内外链自由 | 正文零外链，文末集中 | 外链可保留，但需标注 |

### 1.2 标题优化规则

**网站原版标题**
- 保持原文风格，允许较长、叙事化

**微信公众号标题**（≤ 64 字符）
- 加入情绪词或数字，提高打开率
- 示例：`AI 小队组建日记 Vol.1` → `我造了一支 AI 团队来维护个人网站｜组建日记 Vol.1`

**知乎专栏标题**（≤ 50 字符，推荐 30 以内）
- 强化问题感或方法论
- 示例：`AI 小队组建日记 Vol.1` → `用 AI 团队公开构建个人网站，可行吗？`

### 1.3 段落与格式改写

**通用规则**

| 网站元素 | 微信公众号处理 | 知乎专栏处理 |
|----------|----------------|--------------|
| Markdown 表格 | 转为文字描述或简化列表 | 保留表格，但加文字导读 |
| 代码块 | 截图或简化描述，避免排版崩坏 | 保留代码块，加语言标注 |
| 引用块 `>` | 保留，改为微信编辑器「引用」样式 | 保留 Markdown 引用 |
| 内链（站内文章） | 删除链接，改为文末「相关阅读」 | 改为知乎站内搜索提示或外链 |
| 外链（外部资源） | 删除链接，保留文字，文末集中 | 保留链接，加 `[原文]` 标注 |
| 粗体 `**` | 保留，微信编辑器加粗 | 保留 Markdown 粗体 |
| 层级标题 `##` | 保留，微信用「小标题」样式 | 保留 Markdown 标题 |
| 图片 | 单独上传微信素材库，加图注 | 单独上传知乎图床，加图注 |

**微信公众号特殊处理**
- 每 3–4 段插入一个视觉分隔（空行或分隔符 `---` 的微信等效样式）
- 首段控制在 150 字以内，快速给出"这是什么"
- 避免连续两段都超过 5 行
- 文末固定格式：作者简介 + 引导关注 + 相关阅读卡片

**知乎专栏特殊处理**
- 首段前加 100 字以内的"TL;DR"或核心结论
- 允许比微信更长的技术论证和代码展示
- 文末加 "Edit: " 区域记录后续更新
- 主动在文中埋入 1–2 个可讨论的开放问题

### 1.4 图片处理规则

1. **源图获取**：从网站 `public/` 目录或文章内引用路径获取原始素材
2. **尺寸适配**：
   - 微信：宽度 900px，单张 ≤ 5MB，推荐 PNG/JPG
   - 知乎：宽度 1200px，单张 ≤ 20MB
3. **图注格式**：
   - 微信：图片下方居中，灰色小字（字号 12–13）
   - 知乎：图片下方加 Markdown 斜体图注 `*图：描述*`
4. **GIF/动图**：微信支持 ≤ 300 帧；知乎支持 ≤ 10MB
5. **二维码/外链图**：微信允许文末放公众号二维码；知乎允许放个人主页链接

### 1.5 外链处理规则

**微信公众号**
- 正文内所有 URL 必须删除，改为纯文字描述
- 文末「延伸阅读」区块集中列出最多 5 条链接
- 格式：`📎 相关链接：文章标题（长按识别或搜索）`
- 无法插入可点击外链时，引导读者在站内搜索关键词

**知乎专栏**
- 外链保留，但需判断可信度：
  - 权威来源（GitHub、官方文档）：直接保留
  - 个人博客/小众站点：加 `[非官方来源]` 提示
- 同一篇文章外链不超过 8 条
- 文末加 `本文首发于 PeterClaw 网站，转载请联系授权`

## 2. 发布 Checklist

### 2.1 发布前自检

- [ ] 标题符合平台长度限制，且包含核心关键词
- [ ] 正文无外链残留（微信）/ 外链已标注（知乎）
- [ ] 图片已上传平台图床，图注完整
- [ ] 代码块已适配（微信截图，知乎保留）
- [ ] 表格已转为适配格式
- [ ] 首段 150 字内给出核心价值（微信）/ TL;DR 已加（知乎）
- [ ] 文末 CTA 完整（关注/评论/跳转）
- [ ] 无排版错误（缩进、空行、特殊字符）

### 2.2 话题标签策略

**微信公众号**
- 文章标签（微信后台）：`AI`、`软件开发`、`团队协作`、`个人成长`
- 合集标签：加入「AI 小队组建日记」合集

**知乎专栏**
- 话题：`人工智能`、`AIGC`、`软件工程`、`团队协作`、`个人开发者`
- 问题绑定：如有相关问题，优先回答后插入文章链接

### 2.3 最佳发布时间

| 平台 | 工作日最佳 | 周末最佳 | 避免时段 |
|------|-----------|----------|----------|
| 微信公众号 | 12:00–13:00、21:00–22:00 | 10:00–11:00、20:00–21:00 | 08:00–09:00（通勤）、15:00–17:00（工作） |
| 知乎专栏 | 11:00–12:00、20:00–22:00 | 10:00–12:00、20:00–23:00 | 凌晨 01:00–07:00 |

**AI 日记系列建议节奏**
- 每篇间隔 ≥ 3 天，避免读者疲劳
- Vol.1 → Vol.2 → Vol.3 按顺序发布，建立连续剧感
- 新 Vol. 发布后，在上一篇评论区置顶预告

### 2.4 发布后动作

- [ ] 在网站原文评论区置顶「本文已同步至微信公众号/知乎专栏」
- [ ] 在微信公众号后台查看阅读完成率，低于 40% 检查是否段落过长
- [ ] 在知乎查看赞同/收藏比，收藏高但赞同低说明价值认可但情绪共鸣不足
- [ ] 收集读者问题，作为下一篇 AI 日记的素材输入
- [ ] 更新 `docs/chinese-content-distribution-sop.md` 中的「实战经验」章节

## 5. 附录

### 5.1 快速参考：Markdown → 微信编辑器映射

```
## 标题      → 微信「小标题」样式（字号 16，加粗）
**粗体**    → 微信「加粗」
`代码`      → 微信「代码」样式或截图
> 引用      → 微信「引用」样式（灰色竖线左侧）
- 列表      → 微信「无序列表」
1. 列表     → 微信「有序列表」
[链接](url) → 删除链接，保留文字（正文）/ 文末集中列出
![图](url)  → 上传至微信素材库插入
```

### 5.2 快速参考：Markdown → 知乎编辑器映射

```
## 标题      → Markdown H2
**粗体**    → Markdown 粗体
`代码`      → Markdown 行内代码 / 代码块
> 引用      → Markdown 引用
- 列表      → Markdown 无序列表
1. 列表     → Markdown 有序列表
[链接](url) → Markdown 链接（自动渲染）
![图](url)  → 知乎图床上传或外链图片
```
