# 中文内容分发 SOP：从 Astro Markdown 到全平台分发

> 适用对象：AI 日记系列及后续中文博客文章
> 覆盖平台：微信公众号、知乎专栏、掘金、V2EX 独立开发者、少数派
> 维护者：Kimi 1号（内容策略）
> 最后更新：2026-05-28

---

## 0. 分发全景图

| 平台 | 定位 | 内容形式 | 受众特征 | 可脚本化程度 |
|------|------|----------|----------|--------------|
| **微信公众号** | 私域触达、社交传播 | 长图文 | 泛科技读者 | 低（需人工排版） |
| **知乎专栏** | 公域搜索、专业讨论 | 长文 | 技术从业者 | 中（Markdown 直发） |
| **掘金** | 开发者社区、技术沉淀 | 技术长文 | 前端/全栈开发者 | 高（Markdown + API） |
| **V2EX 独立开发者** | 社区讨论、冷启动 | 短帖 + 外链 | 独立开发者、产品经理 | 中（API 发帖） |
| **少数派** | 品质内容、生活方式 | 长图文 | 效率工具爱好者 | 低（需人工审核） |

**单次分发目标时长：30 分钟内完成全平台。**

## 1. 内容适配指南

### 1.1 核心原则

| 维度 | 网站原版 | 微信公众号 | 知乎专栏 | 掘金 | V2EX | 少数派 |
|------|----------|------------|----------|------|------|--------|
| **定位** | 永久存档、SEO 可索引 | 私域触达、社交传播 | 公域搜索、专业讨论 | 技术沉淀、开发者社群 | 社区讨论、轻量分享 | 品质内容、工具文化 |
| **读者预期** | 已了解 PeterClaw 背景 | 可能是第一次接触 | 关注技术实现细节 | 关注代码与工程实践 | 关注产品决策与数据 | 关注使用体验与效率 |
| **内容长度** | 无限制 | 最佳 1500–3000 字 | 最佳 2000–5000 字 | 最佳 2000–4000 字 | 150–400 字摘要 | 最佳 2000–4000 字 |
| **链接策略** | 站内外链自由 | 正文零外链，文末集中 | 外链可保留，但需标注 | 外链自由，代码仓库优先 | 必须带原文链接 | 外链自由，注重排版 |

### 1.2 标题优化规则

**网站原版标题**
- 保持原文风格，允许较长、叙事化

**微信公众号标题**（≤ 64 字符）
- 加入情绪词或数字，提高打开率
- 示例：`AI 小队组建日记 Vol.1` → `我造了一支 AI 团队来维护个人网站｜组建日记 Vol.1`

**知乎专栏标题**（≤ 50 字符，推荐 30 以内）
- 强化问题感或方法论
- 示例：`AI 小队组建日记 Vol.1` → `用 AI 团队公开构建个人网站，可行吗？`

**掘金标题**（≤ 50 字符）
- 突出技术栈或实战成果，可带 emoji
- 示例：`AI 小队组建日记 Vol.1` → `🤖 5 个 AI Agent 协作维护网站，我们是这样做的`

**V2EX 标题**（≤ 100 字符，但推荐简短）
- 像帖子标题，带结论或问题
- 示例：`分享我用 5 个 AI Agent 公开构建个人网站的实践记录`

**少数派标题**（≤ 40 字符）
- 品质感，避免过度营销
- 示例：`AI 小队组建日记：当五个 Agent 共同维护一个网站`

### 1.3 段落与格式改写

**通用规则**

| 网站元素 | 微信公众号 | 知乎专栏 | 掘金 | V2EX | 少数派 |
|----------|------------|----------|------|------|--------|
| Markdown 表格 | 转文字描述或简化列表 | 保留，加文字导读 | 保留，掘金支持良好 | 不适合表格 | 保留，注意移动端显示 |
| 代码块 | 截图或简化描述 | 保留，加语言标注 | **保留，核心卖点** | 只贴关键 3–5 行 | 保留，但避免大段代码 |
| 引用块 `>` | 保留，改微信「引用」样式 | 保留 Markdown 引用 | 保留 | 不适合 | 保留 |
| 内链（站内） | 删除，改文末「相关阅读」 | 改知乎搜索提示或外链 | 保留为绝对链接 | 保留为原文链接 | 保留 |
| 外链 | 删除，保留文字，文末集中 | 保留，加 `[原文]` 标注 | 自由保留 | 必须保留原文链接 | 自由保留 |
| 粗体 `**` | 保留，微信加粗 | 保留 | 保留 | 可用 `**` | 保留 |
| 层级标题 `##` | 保留，微信「小标题」样式 | 保留 | 保留 | 最多二级 | 保留 |
| 图片 | 上传素材库，加图注 | 上传知乎图床 | 上传掘金图床 | 1–2 张即可 | 上传少数派图床 |

**微信公众号特殊处理**
- 每 3–4 段插入一个视觉分隔（空行或分隔符 `---` 的微信等效样式）
- 首段控制在 150 字以内，快速给出"这是什么"
- 避免连续两段都超过 5 行
- 文末固定格式：作者简介 + 引导关注 + 相关阅读卡片

**知乎专栏特殊处理**
- 首段前加 100 字以内的"TL;DR"或核心结论
- 允许比微信更长的技术论证和代码展示
- 文末加 "Edit: " 区域记录后续更新
- 主动在文中埋入 1–2 个可讨论的开放问题

**掘金特殊处理**
- 开头必须放「阅读约 X 分钟」提示
- 代码块必须标注语言，且建议附加「可运行」或「可复制」提示
- 技术概念首次出现加链接到掘金小册或官方文档
- 文末加「如果对你有帮助，欢迎点赞/收藏/关注」
- 文章底部固定放 GitHub 仓库链接和 Twitter/X 账号

**V2EX 特殊处理**
- 不是完整转载，而是「分享 + 钩子」形式
- 第一段：一句话说清文章价值
- 第二段：2–3 个核心观点/数据点
- 第三段：原文链接 + 邀请讨论
- 可带 1 张封面图或关键截图
- 语气像和人聊天，避免「小编体」

**少数派特殊处理**
- 首段给出「这篇文章适合谁、读完能带走什么」
- 强调工具链、效率提升、具体数字
- 图片要求较高：建议封面图 1200×630，内文图宽度 ≥ 800px
- 文末可放「我的其他工具/文章」推荐
- 避免过度技术细节，侧重「为什么用这个方案」

### 1.4 图片处理规则

1. **源图获取**：从网站 `public/` 目录或文章内引用路径获取原始素材
2. **尺寸适配**：
   - 微信：宽度 900px，单张 ≤ 5MB，推荐 PNG/JPG
   - 知乎：宽度 1200px，单张 ≤ 20MB
   - 掘金：宽度 1000px，单张 ≤ 10MB，支持 GIF
   - V2EX：宽度任意，单张 ≤ 5MB，通常 1–2 张
   - 少数派：宽度 1200px，单张 ≤ 10MB，封面必须精美
3. **图注格式**：
   - 微信：图片下方居中，灰色小字（字号 12–13）
   - 知乎：图片下方加 Markdown 斜体图注 `*图：描述*`
   - 掘金：图片下方加文字说明，可用 `>` 引用样式
   - V2EX：不需要图注
   - 少数派：图片下方加简短说明
4. **GIF/动图**：微信 ≤ 300 帧；知乎 ≤ 10MB；掘金 ≤ 10MB；少数派 ≤ 5MB
5. **二维码/外链图**：微信允许文末放公众号二维码；知乎允许放个人主页链接；掘金允许放公众号/ GitHub 二维码；V2EX 避免；少数派文末可放

### 1.5 外链处理规则

**微信公众号**
- 正文内所有 URL 必须删除，改为纯文字描述
- 文末「延伸阅读」区块集中列出最多 5 条链接
- 格式：`📎 相关链接：文章标题（长按识别或搜索）`
- 无法插入可点击外链时，引导读者在站内搜索关键词

**知乎专栏**
- 外链保留，但需判断可信度：
  - 权威来源（GitHub、官方文档）：直接保留
  - 个人博客/小众站点：加 `[非官方来源]` 提示
- 同一篇文章外链不超过 8 条
- 文末加 `本文首发于 PeterClaw 网站，转载请联系授权`

**掘金**
- 外链自由，但优先使用 GitHub、官方文档、掘金站内链接
- 个人网站链接放在文末「关于作者」区块
- 代码引用尽量指向具体文件行号

**V2EX**
- 必须包含原文链接（`https://peterclaw.com/zh/blog/...`）
- 可附带 GitHub 仓库链接
- 避免短链接，使用完整 URL

**少数派**
- 外链自由，注重排版美观
- 个人网站链接放在文末
- 可引用少数派 Matrix 作者主页

## 2. 发布 Checklist

### 2.1 发布前自检（通用）

- [ ] 标题符合平台长度限制，且包含核心关键词
- [ ] 正文无外链残留（微信）/ 外链已标注（知乎）/ 外链自由（掘金/少数派）/ 原文链接已放（V2EX）
- [ ] 图片已上传平台图床，图注完整
- [ ] 代码块已适配（微信截图，知乎/掘金/少数派保留）
- [ ] 表格已转为适配格式（微信/V2EX 转文字）
- [ ] 首段 150 字内给出核心价值（微信）/ TL;DR 已加（知乎）/ 阅读时间提示已加（掘金）/ 钩子已写（V2EX）/ 适合谁已说明（少数派）
- [ ] 文末 CTA 完整（关注/评论/跳转/原文链接）
- [ ] 无排版错误（缩进、空行、特殊字符）
- [ ] 标签/话题已选对（各平台要求不同，见下文）

### 2.2 话题标签策略

**微信公众号**
- 文章标签（微信后台）：`AI`、`软件开发`、`团队协作`、`个人成长`
- 合集标签：加入「AI 小队组建日记」合集

**知乎专栏**
- 话题：`人工智能`、`AIGC`、`软件工程`、`团队协作`、`个人开发者`
- 问题绑定：如有相关问题，优先回答后插入文章链接

**掘金**
- 分类：`前端`、`人工智能`、`后端`、`架构`
- 标签（最多 5 个）：`AI`、`Astro`、`独立开发`、`开源`、`自动化`
- 系统标签：自动识别，人工复核

**V2EX**
- 节点：`独立开发者`（creators）
- 无需标签，标题即分类

**少数派**
- 分类：`效率工具`、`编程与开发`、`数字生活`
- 标签：文章关键词，如 `AI 工具`、`Astro`、`自动化`

### 2.3 最佳发布时间

| 平台 | 工作日最佳 | 周末最佳 | 避免时段 |
|------|-----------|----------|----------|
| 微信公众号 | 12:00–13:00、21:00–22:00 | 10:00–11:00、20:00–21:00 | 08:00–09:00（通勤）、15:00–17:00（工作） |
| 知乎专栏 | 11:00–12:00、20:00–22:00 | 10:00–12:00、20:00–23:00 | 凌晨 01:00–07:00 |
| 掘金 | 09:00–10:00、14:00–15:00、20:00–21:00 | 10:00–12:00 | 凌晨 01:00–07:00 |
| V2EX | 09:00–10:00、14:00–15:00、21:00–23:00 | 全天均可 | 无特别禁忌 |
| 少数派 | 10:00–11:00、20:00–21:00 | 10:00–12:00 | 工作日 14:00–17:00 |

**AI 日记系列建议节奏**
- 每篇间隔 ≥ 3 天，避免读者疲劳
- Vol.1 → Vol.2 → Vol.3 按顺序发布，建立连续剧感
- 新 Vol. 发布后，在上一篇评论区置顶预告

### 2.4 发布后动作

- [ ] 在网站原文评论区置顶「本文已同步至微信公众号/知乎专栏/掘金/V2EX/少数派」
- [ ] 在微信公众号后台查看阅读完成率，低于 40% 检查是否段落过长
- [ ] 在知乎查看赞同/收藏比，收藏高但赞同低说明价值认可但情绪共鸣不足
- [ ] 在掘金查看阅读/点赞/评论数据，关注「沸点」和「推荐」效果
- [ ] 在 V2EX 跟踪讨论回复，及时参与关键评论
- [ ] 在少数派查看推荐状态（如进入首页推荐则转发到社交媒体）
- [ ] 收集读者问题，作为下一篇 AI 日记的素材输入
- [ ] 更新 `docs/chinese-content-distribution-sop.md` 中的「实战经验」章节

## 3. 可脚本化步骤识别

以下步骤可部分或完全自动化，是未来构建 `scripts/export-content.mjs` 的输入：

| 步骤 | 当前状态 | 脚本化潜力 | 实现优先级 |
|------|----------|------------|------------|
| 从 frontmatter 提取标题、摘要、标签 | 手动 | **高** — 直接读取 Markdown | P0 |
| 生成各平台适配标题（5 个变体） | 手动 | **中** — LLM 改写 | P1 |
| 将 Markdown 表格转为文字列表（微信/V2EX） | 手动 | **高** — 正则 + 模板 | P0 |
| 删除正文内链、保留文字（微信） | 手动 | **高** — Markdown AST 遍历 | P0 |
| 将站内链接转为绝对链接 | 手动 | **高** — 前缀替换 | P0 |
| 生成 V2EX 短帖摘要（150–400 字） | 手动 | **中** — LLM 摘要 | P1 |
| 生成掘金「阅读时间」提示 | 手动 | **高** — 字数统计 ÷ 300 | P0 |
| 生成各平台文末 CTA 区块 | 手动 | **高** — 模板替换 | P0 |
| 提取文章中所有图片路径并批量上传 | 手动 | **中** — 各平台 API 不同 | P2 |
| 一键发布到掘金（API） | 手动 | **高** — 掘金开放 API | P1 |
| 一键发布到 V2EX（API） | 手动 | **中** — V2EX API 需 token | P1 |
| 微信公众号排版 | 手动 | **低** — 微信无开放 API | P3 |
| 少数派投稿 | 手动 | **低** — 需人工审核 | P3 |

## 6. 附录

### 6.1 快速参考：Markdown → 各平台映射

```
## 标题      → 微信「小标题」/ 知乎 H2 / 掘金 H2 / V2EX 粗体 / 少数派 H2
**粗体**    → 微信「加粗」/ 知乎 粗体 / 掘金 粗体 / V2EX 粗体 / 少数派 粗体
`代码`      → 微信 截图 / 知乎 代码块 / 掘金 代码块 / V2EX 行内代码 / 少数派 代码块
> 引用      → 微信「引用」/ 知乎 引用 / 掘金 引用 / V2EX 不适用 / 少数派 引用
- 列表      → 微信「无序列表」/ 知乎 列表 / 掘金 列表 / V2EX 文字 / 少数派 列表
[链接](url) → 微信 删除 / 知乎 保留 / 掘金 保留 / V2EX 保留 / 少数派 保留
![图](url)  → 微信 素材库 / 知乎 图床 / 掘金 图床 / V2EX 上传 / 少数派 图床
```

### 6.2 30 分钟分发流程（建议时序）

```
T+0min   网站文章发布（draft: false → build → deploy）
T+2min   复制原文到掘金模板，调整标题 + 阅读时间提示，上传图片，发布
T+8min   复制原文到知乎模板，加 TL;DR，调整外链标注，发布
T+15min  复制原文到少数派模板，调整首段 + 图片，提交审核
T+20min  用 V2EX 模板写 200 字摘要帖，带原文链接，发布
T+22min  打开微信编辑器，粘贴微信模板版本，手动排版
T+30min  微信预览 → 确认 → 定时发送
```

**优化建议**：掘金、知乎、V2EX 可并行；微信排版耗时最长，放最后；少数派审核非即时，提前提交。
