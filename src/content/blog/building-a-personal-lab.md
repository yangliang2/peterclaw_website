A personal website is most useful when it is easy to update. Instead of treating
this space as a finished brochure, I want it to behave like a small lab: a place
where ideas can become notes, notes can become projects, and projects can leave
behind enough context for someone else to understand the work.

That changes the design priorities. The layout should be quiet, fast, and direct.
The writing should explain the reasoning behind each build. The project archive
should favor useful constraints over decorative case-study language.

This first version is intentionally compact. It gives the site a dark visual
system, a few representative projects, and a publishing path for longer notes.
From here, every iteration can add more signal without replacing the foundation.
