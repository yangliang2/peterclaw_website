> **AI 工具评测专栏 · 第 7 篇**
>
> 当 AI Agent 从「聊天框」走进「工作流」，自动化不再只是「如果 A 则 B」——而是让 AI 替你思考、决策、执行。n8n 的「开源自由」、Make 的「视觉 power」、Zapier 的「连接广度」，谁更适合你的业务？

---

## 评测背景

2026 年，AI 工作流自动化已经从一个「极客玩具」变成了独立开发者和中小企业的「基础设施」。n8n 凭借开源和自托管在开发者社区横扫热度，Make（前身 Integromat）以视觉画布和复杂逻辑著称，Zapier 则依然是连接应用数量最多的老牌霸主。

这三款工具代表了三种截然不同的自动化哲学：

- **n8n** → 开源可自托管的「开发者自动化平台」，AI Agent 工作流是其杀手锏
- **Make** → 视觉画布驱动的「复杂逻辑引擎」，数据转换和分支能力是行业标杆
- **Zapier** → 连接广度最强的「SaaS 自动化入口」，零门槛上手是其核心竞争力

中文社区对这三款工具的讨论很多，但大多停留在「功能介绍」或「价格对比」层面，缺少在真实业务场景中的横向对比。本次评测正是为了填补这个空白。

本次评测基于以下环境：

- **测试周期**：2026-05-12 至 2026-05-25（两周）
- **测试场景**：
  1. 搭建一个「新用户注册 → 欢迎邮件 → CRM 同步 → Slack 通知」的 onboarding 自动化（3 个工具各实现一遍）
  2. 构建一个 AI 内容摘要工作流：抓取 RSS → LLM 生成摘要 → 存入 Notion → 异常时人工审核（含 AI Agent 节点测试）
  3. 高复杂度分支逻辑：电商订单路由（根据金额、地区、库存状态走不同分支）
  4. 稳定性测试：连续 7 天运行，观察失败率、延迟和异常恢复能力
- **系统环境**：macOS 15.4，同时测试了 n8n 自托管（Docker on Hetzner CX22，€4.51/月）
- **订阅状态**：n8n Cloud Starter（€24/月）、Make Core（$9/月）、Zapier Professional（$29.99/月），均为个人付费，无利益相关

目标读者：正在用手动操作或简单脚本处理重复工作，考虑升级到工作流自动化工具的独立开发者、运营人员和中小企业主。

---

## 评测维度

本次评测聚焦六个核心维度：

1. **价格与免费额度**——订阅费用、免费版可用性、规模化成本
2. **自定义节点能力**——能否对接内部 API、写自定义代码、扩展平台边界
3. **AI 节点集成**——LLM 调用、向量数据库、AI Agent 工作流的支持深度
4. **社区与模板生态**——官方模板数量、社区活跃度、第三方集成丰富度
5. **上手难度**——从 0 到第一个自动化上线所需的时间和技能门槛
6. **稳定性与可观测性**——长时间运行的失败率、错误恢复、调试体验

## n8n：概述

n8n 是一款 fair-code 开源工作流自动化平台，2019 年发布，2024-2025 年凭借 AI Agent 节点和自托管优势在开发者社区爆发式增长。它的核心卖点是「开源 + 自托管 + 深度 AI 集成」——你可以在本地服务器上免费运行无限量的工作流，同时原生支持 LangChain 风格的 AI Agent。

### 优点

**自托管 = 无限自由 + 极致性价比。** n8n 的 Community Edition 完全免费，支持无限工作流、无限执行次数、无限用户。我在 Hetzner 上花 €4.51/月租了一台 CX22 VPS，Docker 部署 n8n 后稳定运行了两周，处理了约 3,000 次工作流执行，成本几乎为零。对比 Zapier，同样的执行量在 Professional 计划（$29.99/月，750 tasks）下根本不够看——如果按 Zapier 的 task 计费模型，3,000 次执行 × 平均 5 个步骤 = 15,000 tasks，需要 Team 计划（$103.50/月，2,000 tasks 仍不够，实际可能要 $300+/月）。

**AI Agent 工作流是三款中最强的。** n8n 原生集成了 70+ 个 AI 相关节点，覆盖 OpenAI、Anthropic Claude、Google Gemini、DeepSeek 等 15+ 个 LLM 提供商，以及 Pinecone、Qdrant、Weaviate、Supabase pgvector 等向量数据库。在测试「RSS → AI 摘要 → Notion」工作流时，我不仅调用了 GPT-4o 生成摘要，还用一个 Vector Store 节点将历史摘要存入 Qdrant，实现了「基于历史内容的个性化摘要风格」——这种 RAG + Agent 的完整链路，在 Make 和 Zapier 中需要大量拼接才能实现。

**自定义能力几乎没有天花板。** n8n 的 Code 节点支持原生 JavaScript 和 Python，你可以直接在工作流中写复杂的数据转换逻辑。更强大的是，n8n 支持开发自定义节点——我用了一个下午就封装了一个调用内部 API 的自定义节点，提交到社区后还可以被他人复用。这种「平台即框架」的开放性，是闭源的 Make 和 Zapier 无法比拟的。

**执行计费模型对复杂工作流极其友好。** n8n 按「整个工作流执行次数」计费，而不是按步骤。一个包含 20 个节点的复杂工作流运行一次，只算 1 次执行。在测试中，我的电商订单路由工作流有 12 个节点，在 n8n 中每次运行成本几乎为零（自托管），而在 Zapier 中同样的工作流会消耗 12 个 tasks。

### 缺点

**上手门槛是三款中最高的。** 要部署自托管 n8n，你需要：准备 VPS → 安装 Docker → 配置环境变量 → 设置反向代理（Nginx/Caddy）→ 配置 SSL。对于非技术用户，这个流程直接劝退。即使是 n8n Cloud，其界面中的「表达式」（expressions）和「节点配置」对没有编程背景的用户来说也不够直观。

**自托管意味着运维责任在你身上。** 在测试期间，我遇到了一次 n8n 容器 OOM（内存不足）导致服务重启，虽然数据没有丢失，但正在执行的工作流被中断了。如果你不愿意处理 Docker、日志、备份、SSL 证书续期等问题，n8n 的自托管优势就变成了负担。

**移动端体验几乎不存在。** n8n 没有原生移动 App，手机端只能通过浏览器访问 Web 界面，在小屏幕上操作非常痛苦。Make 和 Zapier 都有相对成熟的移动端体验，至少可以查看执行日志和启停工作流。

**Cloud 计划的定价跳跃较大。** Starter（€24/月，2,500 执行）到 Pro（€60/月，10,000 执行）的跨度对快速增长的小团队来说有点尴尬。如果你每月需要 5,000 次执行，要么花 €60 买 Pro，要么自托管——中间没有过渡选项。

## Make：概述

Make（原名 Integromat）是 2016 年推出的视觉自动化平台，2022 年完成品牌升级。它的核心卖点是「视觉画布 + 复杂逻辑」——你可以在一个无限大的画布上拖拽模块，构建带有循环、分支、聚合、错误处理的多步工作流。对于需要复杂数据转换和条件路由的场景，Make 是三款中表达力最强的。

### 优点

**视觉画布让复杂逻辑一目了然。** Make 的 scenario builder 是我用过的最直观的视觉工作流设计器。在测试电商订单路由时，我用一个画布就画出了「金额 > ¥500 → 高端客服」、「库存 < 10 → 预警通知」、「海外订单 → 关税计算」三条分支，每条分支还有自己的子分支。这种复杂度在 Zapier 的线性步骤界面中几乎无法优雅表达，在 n8n 中虽然也能实现，但视觉化程度不如 Make。

**数据处理能力最强。** Make 内置了极其丰富的数据转换函数：字符串操作、JSON 解析、数组映射、日期计算、正则表达式……在测试中，我需要将一个嵌套 JSON 的订单数据扁平化后写入 Google Sheets，Make 的「Iterator + Aggregator」组合在 3 分钟内就搞定了。同样的操作在 Zapier 中需要多个 Formatter 步骤，在 n8n 中需要写 JavaScript。

**价格是 Zapier 的 1/3，免费版更慷慨。** Make Core 计划 $9/月（年付）提供 10,000 个 operations，免费版也有 1,000 operations/月。对比 Zapier Professional $29.99/月只有 750 tasks，Make 的性价比优势明显。更重要的是，Make 的所有应用在所有付费计划中都可以使用（不像 Zapier 把某些应用标为 Premium，需要更高计划）。

**实时触发而非轮询。** Make 支持大量应用的实时 webhook 触发，而 Zapier 的免费和入门计划很多触发器是「每 15 分钟轮询一次」。在测试新用户注册自动化时，Make 的 webhook 触发几乎是瞬时的，Zapier 的轮询则平均延迟 7-8 分钟（免费版更慢）。

### 缺点

**AI 集成深度明显弱于 n8n。** Make 确实有 OpenAI、Claude、Gemini 的 AI 模块，也推出了 Maia AI 助手和 Make AI Agents，但这些功能更像是「在视觉工作流中调用 AI API」，而不是 n8n 那种「原生 AI Agent 架构」。在测试 RAG 工作流时，Make 没有原生向量数据库节点，需要手动调用 Pinecone API 并用 HTTP 模块拼接——整个过程比 n8n 多了 5-6 个模块，维护成本更高。

**操作计费模型对多步工作流不友好。** Make 按 operation（模块执行次数）计费，一个包含 10 个模块的 scenario 运行一次 = 10 operations。如果你有一个高频触发的工作流（比如每分钟接收一次 webhook），且工作流本身有 8 个模块，那么每天就是 10 × 60 × 24 = 14,400 operations——免费版和 Core 计划的 10,000 operations 很快会被耗尽。

**无自托管选项，数据必须上云。** Make 是纯 SaaS，没有自托管版本。对于数据合规要求严格的行业（金融、医疗、政务），这意味着所有业务数据都必须经过 Make 的服务器。相比之下，n8n 的自托管可以让你完全控制数据流向。

**学习曲线比 Zapier 陡。** Make 的视觉画布虽然强大，但新手第一次打开时会被满屏的模块、连接线、分支逻辑吓到。我在测试时让一位没有技术背景的运营同事尝试搭建一个简单的「表单 → 邮件」自动化，她花了 45 分钟才完成——同样的任务在 Zapier 中只花了 10 分钟。

## Zapier：概述

Zapier 是 2011 年推出的工作流自动化平台，也是这个赛道的开创者和绝对霸主。它的核心卖点是「连接最多应用 + 零门槛上手」——拥有 7,000+ 应用集成，几乎你能想到的 SaaS 工具都能在 Zapier 中找到。对于非技术用户来说，Zapier 依然是「第一个自动化」的最佳选择。

### 优点

**应用集成数量无人能敌。** Zapier 的 7,000+ 应用目录是 Make（~2,000）和 n8n（500+ 官方节点）的数倍。在测试中，我需要连接一个相对小众的邮件营销工具（MailerLite），Zapier 有官方集成，Make 有但功能不完整，n8n 没有官方节点只能通过 HTTP 请求手动对接。对于依赖大量小众 SaaS 工具的业务，Zapier 的集成广度是决定性优势。

**上手门槛几乎为零。** Zapier 的界面设计是三款中最「傻瓜化」的：选择触发应用 → 选择触发事件 → 选择动作应用 → 选择动作 → 映射字段。我的运营同事第一次使用 Zapier，10 分钟内就搭建了一个「Google Forms 新回复 → 钉钉群通知」的自动化。这种「零学习成本」的体验，是 Zapier 在中小企业市场屹立不倒的核心原因。

**Zapier Copilot 让 AI 真正降低了使用门槛。** 2025-2026 年，Zapier 推出了 Copilot 功能——你可以用自然语言描述想要的自动化（比如「当有新订单时，给仓库发邮件并更新库存表」），AI 会自动推荐合适的应用组合并预填字段。在测试中，我用中文描述了一个 3 步自动化，Copilot 在 30 秒内生成了完整的 Zap，字段映射准确率约 80%。对于完全不懂自动化的用户，这个功能意义重大。

**生态系统成熟，周边工具丰富。** Zapier 不仅有自动化，还推出了 Tables（数据库）、Interfaces（前端界面）、Canvas（自动化架构图）等周边产品。虽然单个产品都不算顶尖，但「全家桶」的组合对于希望在一个平台解决多个问题的中小企业很有吸引力。

### 缺点

**价格是三款中最贵的，且规模化成本极高。** Zapier Professional $29.99/月只有 750 tasks，Team $103.50/月只有 2,000 tasks。更糟糕的是，Zapier 按 task（每个动作步骤）计费——一个 5 步 Zap 运行 100 次 = 500 tasks。在测试的电商订单路由场景中，同样的工作流在 Zapier 中每月成本约为 Make 的 4-5 倍、n8n 自托管的 50 倍以上。

**复杂逻辑支持薄弱。** Zapier 的线性步骤设计在简单自动化时很优雅，但一旦涉及复杂分支、循环或错误处理，体验急剧下降。虽然 Professional 以上计划支持 Paths（条件分支）和 Loop，但这些功能像是「后期补丁」，远不如 Make 的画布原生优雅，也不如 n8n 的节点自由组合灵活。

**AI Agent 能力是付费附加功能。** Zapier 的 AI Agents 功能需要额外支付 $33.33/月（在 Professional 计划之上），这让入门成本直接飙升到 $63/月。对比 n8n（AI 节点免费使用，只收 LLM API 费用）和 Make（AI 模块包含在付费计划中），Zapier 的 AI 策略显得过于保守。

**定制能力几乎为零。** Zapier 不支持自定义代码节点（虽然有 Code by Zapier，但功能极其有限），也不支持开发自定义应用集成。如果你的业务需要对接内部系统或小众 API，Zapier 的「封闭花园」会成为瓶颈。

## 场景化推荐

### 如果你是一名独立开发者或技术型创始人，注重成本控制和数据主权

**推荐 n8n（自托管）。**

独立开发者的核心诉求是「花最少的钱，获得最大的控制力」。n8n 自托管的 €5/月 VPS 成本，加上免费的无限工作流和原生 AI Agent 能力，是其他工具无法比拟的。你可以对接内部 API、写自定义代码节点、甚至把向量数据库和本地 LLM（Ollama）跑在同一台服务器上。数据完全在你手中，没有 vendor lock-in。

### 如果你是一名运营人员或市场经理，需要处理复杂的数据转换和分支逻辑

**推荐 Make。**

运营工作的特点是「数据源多、格式杂、规则复杂」。Make 的视觉画布和数据处理函数让这种复杂性变得可管理——你可以清楚地看到数据如何从 A 流到 B，在哪个节点被转换，在哪个分支被过滤。$9/月的 Core 计划对大多数中小型运营团队来说已经足够。

### 如果你是一名非技术背景的中小企业主，希望最快上线第一个自动化

**推荐 Zapier。**

对于「我只想自动化一件事，不想学习任何新东西」的用户，Zapier 依然是最佳选择。7,000+ 应用集成意味着你用的工具几乎都能连上，Copilot 的 AI 辅助让搭建过程像聊天一样简单。虽然价格较贵，但如果你每月只需要处理几百个 tasks，Professional 计划的成本是可以接受的。

### 如果你正在构建 AI 原生工作流（RAG、Agent、记忆系统）

**推荐 n8n。**

AI 工作流不仅仅是「调用 OpenAI API」，而是需要向量存储、工具调用、记忆管理、多 Agent 协作的完整链路。n8n 的 AI Agent 节点、LangChain 集成和原生向量数据库支持，让你可以在一个平台内构建生产级的 AI 自动化。这是 Make 和 Zapier 目前无法企及的深度。

### 如果你在一个数据合规要求严格的行业（金融、医疗、政务）

**推荐 n8n（自托管）或 Zapier Enterprise。**

数据不出境是硬要求。n8n 自托管可以让你把数据完全留在本地服务器或私有云上，满足 GDPR、等保等合规要求。如果团队完全非技术，Zapier Enterprise 提供自定义数据驻留选项，但价格要高得多。

### 如果你需要连接大量小众或垂直行业 SaaS 工具

**推荐 Zapier。**

Zapier 的 7,000+ 应用目录覆盖了大量长尾 SaaS。如果你用的工具在 n8n 和 Make 中都没有官方集成，Zapier 很可能是唯一的选择。虽然可以通过 n8n 的 HTTP 节点或 Make 的 Webhook 手动对接，但维护成本会显著增加。

---

## 最终结论

> **综合推荐：★★★★★**
>
> n8n 是「技术团队 + AI 工作流」的最优解，Make 是「复杂业务逻辑 + 视觉设计」的强力选手，Zapier 是「非技术用户 + 快速连接」的默认选择。三款工具没有绝对的胜负，只有「你的技术栈和团队能力更适合哪种哲学」。

**一句话总结**：

- 如果你相信「自动化应该完全可控、成本趋零、AI 原生」→ **n8n（自托管）**
- 如果你相信「自动化的可视化设计比代码更重要」→ **Make**
- 如果你相信「最快连上最多应用就是最大价值」→ **Zapier**

**未来观望点**：

- n8n 在 2026 年 Q2 加入了 MCP（Model Context Protocol）支持，这意味着它可以连接越来越丰富的 AI 工具生态——未来 n8n 可能不只是自动化平台，而是「AI 工具编排中心」
- Make 正在内测更强大的 AI Agents 功能，如果正式发布时向量数据库支持到位，可能缩小与 n8n 在 AI 工作流上的差距
- Zapier 的 Copilot 正在快速迭代，如果 AI 生成工作流的准确率提升到 95% 以上，可能重新定义「零门槛自动化」的标杆
- 三款工具都在探索「自然语言 → 工作流」的生成路径——未来的竞争可能不是「谁的画布更好用」，而是「谁更能听懂人话」
