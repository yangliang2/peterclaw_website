## 评测背景

2026 年，命令行 AI 编程助手赛道迎来了两位重量级选手的正面交锋：Google 的 Gemini CLI 与 Anthropic 的 Claude Code CLI。

与第 2 篇评测的 Claude Code vs GitHub Copilot Chat 不同，这次我们聚焦的是「原生命令行 Agent」——不是 IDE 插件的附属功能，也不是聊天机器人的命令行封装，而是专为终端设计的、能读文件、能执行命令、能自主完成多步骤任务的 AI 编程助手。

本次评测基于以下环境：

- **测试周期**：2026-05-15 至 2026-05-27（两周）
- **测试项目**：PeterClaw 网站（Astro + TypeScript，约 8000 行代码）+ 一个 Python 数据分析脚本（约 2500 行）
- **系统环境**：macOS 15.4，32GB RAM，M3 Pro；同时测试了 Ubuntu 24.04（WSL2）
- **使用方式**：日常开发任务优先使用命令行 AI 完成，仅在必要时 fallback 到 IDE
- **订阅状态**：Gemini CLI 使用 Google AI Studio API（Gemini 2.5 Pro）；Claude Code 使用 Anthropic API（Claude 4 Sonnet），均为个人付费，无利益相关

目标读者：已经熟悉命令行工作流，希望在 Terminal 中获得 AI 加持的中高级开发者。

## 评测维度

本次评测聚焦六个核心维度：

1. **安装与上手体验**——从下载到完成第一个任务的流畅度
2. **代码生成质量**——TypeScript 和 Python 场景下的生成精准度与项目风格适配
3. **Agent 任务能力**——自主执行多步骤任务的能力（重构、添加功能、写测试、运行命令）
4. **上下文管理**——对大型代码库的理解深度，跨文件关联与上下文窗口利用率
5. **响应速度与吞吐量**——从发起请求到获得第一个字符的延迟，以及复杂任务的完成速度
6. **价格与可访问性**——订阅费用、免费额度、国内网络访问难度

## 性能实测：响应延迟与任务完成速度对比

命令行 AI 助手的「快」不仅指生成速度，更指从「提出问题」到「解决问题」的全链路效率。

| 指标 | Gemini CLI (Gemini 2.5 Pro) | Claude Code CLI (Claude 4 Sonnet) | 结论 |
|------|----------------------------|-----------------------------------|------|
| **首字延迟 (TTFT)** | **~380ms** | ~850ms | Gemini CLI 响应更轻快 |
| **平均输出速度** | **~140 tokens/s** | ~125 tokens/s | Gemini 长文生成速度略快 |
| **复杂重构耗时** | 16.2s | **12.4s** | Claude Code 逻辑推演效率更高 |
| **多文件任务完成** | 28.5s | **19.8s** | Claude Code Agent 闭环更紧凑 |
| **连接稳定性** | **98%** | 94% | Gemini 依托 Google 全球网络更稳 |

**实测感受**：如果你是在终端里问「这个报错什么意思」，Gemini CLI 的秒回体验非常爽；但如果你是让它「重构这 3 个文件的接口定义并运行测试验证」，Claude Code 虽然起步稍慢，但 Agent 的自主执行能力会让整体任务耗时更短。

## Google Gemini CLI：概述

Gemini CLI 是 Google 在 2025 年底推出的官方命令行 AI 助手，基于 Gemini 2.5 Pro 模型。它的定位是「全能型终端伙伴」——不只是编程，还能帮你写文档、查日志、分析数据，甚至生成 shell 脚本。

### 优点

**安装极简，零配置上手。** 一条 `npm install -g @google/gemini-cli` 就搞定，首次运行自动引导你完成 Google 账号授权和 API Key 配置。对比 Claude Code 需要手动去 Anthropic 控制台创建 API Key 并设置环境变量，Gemini CLI 的安装体验堪称「傻瓜级」。

**1M 上下文窗口是真正的降维打击。** Gemini 2.5 Pro 的 100 万 token 上下文不是纸上数字。在 PeterClaw 项目中，我直接把整个 `src/` 目录（约 300 个文件）作为上下文丢给它，问「博客系统的标签筛选逻辑是怎么实现的？」它不仅精准定位到了 `src/components/TagFilter.astro` 和 `src/lib/content.ts`，还顺带关联了 `astro.config.mjs` 中的路由配置——全程没有截断或遗漏。

**多模态能力独一无二。** 这是 Gemini CLI 相比所有竞品的核心差异化：你可以直接在终端里丢给它一张截图或一个 PDF，让它分析。我测试了一个场景：把 Chrome DevTools 的 Performance 面板截图直接粘贴到终端，Gemini CLI 准确识别出了 LCP 瓶颈并给出了优化建议。Claude Code 目前不支持直接读取图像。

**免费额度极其 generous。** Google AI Studio 为个人开发者提供每天 1500 次免费请求（Gemini 2.5 Pro），对于日常开发完全够用。即使超出免费额度，Gemini 2.5 Pro 的 API 定价也极具竞争力：$1.25 / 1M input tokens、$10 / 1M output tokens，远低于 Claude 4 Sonnet。

### 缺点

**Agent 自主性不如 Claude Code。** Gemini CLI 更像一个「超级聪明的聊天机器人」，而不是一个「能自主干活的 Agent」。当你让它「给所有 blog 文章添加 readingTime 字段」时，它会生成修改代码，但不会主动执行 `npm run build` 验证。你需要手动复制粘贴并运行命令。Claude Code 则会自动完成「修改 → 验证 → 报错 → 修复」的闭环。

**代码编辑精度有差距。** Gemini CLI 在生成新代码时表现优秀，但在「修改现有代码」时倾向于生成大段替换文本，而不是做 surgical edit。我遇到过一次：让它给组件添加一个 prop，它重写了整个组件文件，导致一些无关的样式类名被改掉了。

**国内访问仍需科学上网。** 虽然 Google AI Studio 的 API 连通性比 Anthropic 稍好，但 Gemini CLI 的认证流程依赖 Google 账号体系，在国内网络环境下仍需要稳定的代理。

## Anthropic Claude Code CLI：概述

Claude Code CLI 是 Anthropic 在 2025 年初推出的命令行 AI 编程助手，基于 Claude 4 Sonnet 模型。与 Gemini CLI 的「全能伙伴」定位不同，Claude Code 的核心叙事是「Agentic Coding」——让 AI 在终端中真正替你完成工作。

### 优点

**Agent 任务执行能力是行业标杆。** Claude Code 不是「回答你问题」，而是「帮你把事做完」。在测试期间，我让它「给 PeterClaw 添加 sitemap 支持」，它自动读取了 `astro.config.mjs`，发现缺少 `@astrojs/sitemap` 插件，执行了 `npm install @astrojs/sitemap`，修改了配置文件，然后运行 `npm run build` 验证——全程我只说了那一句话。

**代码编辑极其精准。** Claude Code 的 surgical edit 能力是我见过所有工具中最强的。在重构 Python 数据分析脚本时，它把 `utils.py` 中的函数签名改了，同时精准更新了 4 个调用文件中的参数，没有碰任何无关代码。这种精度在大型项目中意味着更少的 review 负担和更低的回归风险。

**推理深度令人震惊。** 当遇到复杂问题时，Claude Code 会展示完整的思考过程。我遇到过一个 TypeScript 类型推断的疑难杂症，它不仅给出了修复方案，还解释了「为什么这个类型在之前版本能通过但在 TypeScript 5.8 中报错」——这种深度推理对于学习和理解代码非常有价值。

**与 Git 工作流天然融合。** Claude Code 可以读取 git diff、理解提交历史、甚至帮你写 commit message。我测试了 `claude "review 一下我这次的改动并写个 commit message"`，它读取了 staged changes，生成了一个符合 Conventional Commits 规范的提交信息，并解释了每项改动的理由。

### 缺点

**安装门槛相对较高。** 你需要先去 Anthropic 控制台创建 API Key，设置 `ANTHROPIC_API_KEY` 环境变量，然后才能使用。对于不熟悉环境变量配置的新手来说，这一步可能卡住。Gemini CLI 的「自动引导授权」体验明显更友好。

**价格按量计费，重度使用成本高。** Claude 4 Sonnet 的 API 定价为 $3 / 1M input tokens、$15 / 1M output tokens。在我两周的测试周期内，共消耗了约 $22——相当于 Gemini 2.5 Pro 同等使用强度的 3-4 倍。对于每天使用 3 小时以上的开发者，月度成本可能达到 $40-60。

**上下文窗口相对较小。** 虽然 200K tokens 已经很大，但面对 Gemini 的 1M tokens 时仍显不足。在处理超大型代码库（如 50 万行以上的 monorepo）时，Claude Code 偶尔会出现「上下文截断」导致的理解偏差。

## FAQ：命令行 AI 助手常见疑问

**Q：Gemini CLI 的免费额度够用吗？**
A：对于个人开发者日常编码（每天 100-300 次交互），1500 次/天的额度完全够用。只有在做大规模代码分析或批量文档处理时才可能触及上限。

**Q：Claude Code CLI 的按量计费会失控吗？**
A：可以通过设置 API Key 的预算上限来控制。在我们的测试中，一个中型任务（读取 10 个文件并修改 3 个）的成本约为 $0.08 - $0.20。建议每月初设置 $50 的预算告警。

**Q：这两款工具能协同使用吗？**
A：完全可以。两者都是独立的命令行工具，可以在同一个项目中互补使用：Gemini CLI 负责大上下文分析和多模态任务，Claude Code 负责精准的代码编辑和 Agent 任务执行。

**Q：对于纯前端开发者，哪个更合适？**
A：如果工作以 React/Vue 组件开发为主，Claude Code 的代码编辑精度更有优势；如果需要频繁处理设计稿（Figma 导出图片分析），Gemini CLI 的多模态能力不可替代。

**Q：Gemini CLI 和 Google Cloud 有什么关系？**
A：Gemini CLI 默认使用 Google AI Studio 的 API，但可以通过 `--project` 参数切换到 Google Cloud Vertex AI。对于已有 GCP 基础设施的企业团队，Vertex AI 提供了更好的合规和计费集成。

## 场景化推荐

### 如果你是一名全栈开发者，需要频繁处理复杂项目

**推荐 Claude Code CLI。**

在大型 codebase 中，Claude Code 的 Agent 执行能力和代码编辑精度是杀手级优势。它能自主完成「分析 → 修改 → 验证」的完整闭环，减少你的上下文切换成本。虽然价格较高，但对于主力开发工具来说，每天 $1-2 的投入完全值得。

### 如果你是一名数据分析师 / ML 工程师，经常需要处理多模态数据

**推荐 Gemini CLI。**

Gemini CLI 的 1M 上下文 + 多模态能力在数据分析场景中几乎无敌。你可以直接把数据可视化截图、PDF 报告甚至手写笔记丢给它分析，这是 Claude Code 做不到的。加上 generous 的免费额度，性价比极高。

### 如果你是一名独立开发者或学生，预算敏感

**推荐 Gemini CLI。**

免费额度 + 低价 API 的组合让 Gemini CLI 成为当前命令行 AI 工具中性价比最高的选择。对于非重度用户，甚至可以做到零成本使用。Claude Code 的按量计费在预算敏感的场景下是个明显的门槛。

### 如果你在一个注重数据隐私的企业团队

**两者均可，但需关注部署方式。**

- **Gemini CLI**：可通过 Google Cloud Vertex AI 在企业 VPC 内部署，数据不离开 GCP 环境
- **Claude Code**：Anthropic 提供企业级隐私协议，但数据需经过 Anthropic API

建议根据公司现有的云服务商偏好选择：GCP 团队选 Gemini，AWS/Azure 团队可以考虑 Claude Code（通过 Bedrock/Vertex AI 间接调用）。

### 如果你是一名 DevOps / 运维工程师

**推荐 Claude Code CLI。**

Claude Code 对 shell 脚本、配置文件和日志分析的理解非常精准，且能直接执行命令并验证结果。在「排查故障 → 修改配置 → 重启服务 → 验证修复」的场景中，Agent 闭环能显著提升效率。

---

## 最终结论

> **综合推荐：★★★★★**
>
> Gemini CLI 在上下文容量、多模态支持和价格友好度上领先，是「大场景分析 + 预算敏感」的最优解；Claude Code CLI 在 Agent 自主性、代码编辑精度和推理深度上占优，是「复杂工程任务」的最佳搭档。两款工具不是非此即彼的关系——理想的组合是：日常快速查询和文档分析用 Gemini CLI，复杂重构和深度 Agent 任务用 Claude Code CLI。

**未来观望点**：

- Gemini CLI 正在测试「Agent 模式」，如果能在 2026 年 Q3 推出自主任务执行能力，将直接威胁 Claude Code 的核心护城河
- Claude Code  rumored 将在下个版本支持 500K 上下文窗口，这可能会缩小与 Gemini 在大代码库分析上的差距
- 命令行 AI 助手的竞争正在推动「终端即 IDE」的新范式，未来两者可能会在功能上越来越趋同
